# 项目结构优化计划

## 1. 问题分析

### 1.1 代码重复
- 多个功能相似的训练脚本：`train_model.py`、`train_model_v2.py`、`train_model_improved.py` 等
- 重复的数据处理脚本：`split_dataset.py`、`split_dataset_v2.py` 等
- 重复的评估脚本：`evaluate_model.py` 在多个目录存在
- 标签生成功能分散在不同文件中

### 1.2 目录结构问题
- `arona/` 目录与 `src/models/` 目录功能重叠
- 数据相关脚本分布在多个目录
- 配置文件分散在多个位置
- 文件命名不规范，包含中文或数字

## 2. 优化方案

### 2.1 目录结构优化

```
anime_role_detect/
├── config/             # 统一配置目录
│   ├── character_attributes.json
│   ├── config.json
│   └── user-agent.txt
├── data/               # 数据目录
│   ├── downloaded_images/  # 下载的图片
│   ├── image_tags/         # 生成的标签
│   ├── processed_data/     # 处理后的数据
│   └── raw_data/           # 原始数据
├── models/             # 模型目录
│   ├── arona_plana/        # 阿罗娜普拉娜模型
│   ├── wd-vit-v3/          # WD Vit V3模型
│   └── onnx/               # ONNX格式模型
├── src/                # 源代码目录
│   ├── core/               # 核心功能
│   │   ├── classification/   # 分类相关
│   │   ├── data_processing/  # 数据处理
│   │   ├── tagging/          # 标签生成
│   │   └── utils/            # 工具函数
│   ├── data/               # 数据相关
│   │   ├── collection/       # 数据采集
│   │   └── preprocessing/    # 数据预处理
│   ├── models/             # 模型相关
│   │   ├── training/         # 模型训练
│   │   └── evaluation/       # 模型评估
│   ├── scripts/            # 脚本工具
│   └── web/                # Web相关
├── tests/              # 测试目录
├── README.md
└── requirements.txt
```

### 2.2 代码整合

#### 2.2.1 训练脚本整合
- 创建统一的训练脚本，支持多种模型架构
- 提取公共训练逻辑到核心模块
- 支持命令行参数配置

#### 2.2.2 数据处理脚本整合
- 创建统一的数据处理模块
- 支持多种数据格式和处理方式
- 提供标准化的数据接口

#### 2.2.3 标签生成整合
- 整合WD Vit V3 Tagger功能
- 提供统一的标签生成接口
- 支持多种标签模型

#### 2.2.4 评估脚本整合
- 创建统一的评估脚本
- 支持多种评估指标
- 生成标准化的评估报告

## 3. 实施步骤

### 3.1 第一阶段：目录重组织
1. 创建新的目录结构
2. 移动现有文件到新位置
3. 更新导入路径

### 3.2 第二阶段：代码整合
1. 整合训练脚本
2. 整合数据处理脚本
3. 整合标签生成功能
4. 整合评估脚本

### 3.3 第三阶段：测试和验证
1. 测试所有功能是否正常
2. 验证模型性能是否保持
3. 优化性能和可靠性

## 4. 预期成果

- 减少代码重复，提高代码可维护性
- 统一目录结构，提高项目可读性
- 标准化接口，方便扩展和集成
- 提高开发效率，减少维护成本
